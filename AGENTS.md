# Repository Guidelines

## Project Structure & Module Organization
This repository is a small collection of standalone Python tools for Brazilian fiscal document parsing and bank reconciliation. Key files live in the repo root:

- `brazil_product_code_v1.02.py`: PySide6 desktop app for invoice parsing and batch export.
- `duizhang_gui v1.00.py`: Tkinter GUI for bank statement reconciliation.
- `*.spec`: PyInstaller build definitions for the two apps.
- `test_recognition.py`: Ad-hoc parsing test runner for sample PDFs.
- `*.pdf`: Sample invoices used for manual testing.
- `danfe_data_autosave.json` and `invoice_payment.db`: Local data artifacts used by the tools.

## Build, Test, and Development Commands
Run apps directly with Python:

```bash
python "brazil_product_code_v1.02.py"
python "duizhang_gui v1.00.py"
```

Build Windows executables with PyInstaller:

```bash
pyinstaller "brazil_product_code_v1.02.spec"
pyinstaller "duizhang_gui v1.00.spec"
```

## Coding Style & Naming Conventions
- Use 4-space indentation; avoid tabs.
- Python naming: `lower_snake_case` for functions/variables, `CapWords` for classes, `UPPER_SNAKE_CASE` for constants.
- Follow the existing versioned filename pattern (e.g., `v1.02`) when publishing new release scripts or specs.
- No formatter or linter is currently configured; keep changes consistent with nearby code.

## Testing Guidelines
There is no formal test framework. Use the ad-hoc script to validate invoice parsing on the sample PDFs:

```bash
python test_recognition.py
```

If you add tests, place them in the root alongside `test_recognition.py` and keep names descriptive (e.g., `test_parser_basic.py`).

## Commit & Pull Request Guidelines
This folder does not include Git history. If you initialize a repository, use concise, imperative commit messages (e.g., "Fix OCR normalization"). For pull requests, include:

- A short summary of behavior changes.
- Screenshots or screen recordings for GUI changes.
- Any new sample inputs used for validation (PDFs/JSON), or a note if none were added.

## Data & Configuration Notes
Local data files like `danfe_data_autosave.json` and `invoice_payment.db` are generated by the tools. Avoid committing sensitive or customer data.
